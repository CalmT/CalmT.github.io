CSS编码规范

发表于 2017-04-25 | 分类于 编码规范 | 阅读次数
If you never try, you will never know! 一直想着去接触后端，换了新工作后，终于如愿了，只是没想到还是学校学的asp+sqlserver那套，苍天饶过谁！[耸肩无奈]本文档的目的是使CSS代码风格保持一致，容易被理解和被维护

代码风格

文件编码

[建议]设置css文件编码为utf-8

缩进

[强制] 使用 4 个空格做为一个缩进层级，不允许使用 2 个空格 或 tab 字符。

可以设置编辑器tab键为4个空格

空格

[强制] 选择器 与 ‘{’ 之间、‘:’ 与 属性值 之间必须包含空格

列表型属性值书写在单行时，‘,’ 后必须跟一个空格。

选择器

[强制] 当一个 rule 包含多个 selector 时，每个选择器声明必须独占一行。

[强制]当使用子元素选择器或兄弟选择器时 >、+、~ 选择器的两边各保留一个空格。

[强制] 属性选择器中的值必须用双引号包围。不允许使用单引号

属性声明

[强制]对于只包含一条声明的样式，为了易读性和便于快速编辑，建议将语句放在同一行。对于带有多条声明的样式，还是应当将声明分为多行。

[强制]每条属性声明后必须加分号，即使是最后一条语句

注释格式：/ 注释文字/

对选择器的注释统一写在被注释对象的上一行，对属性及值的注释写于分号后。

注释内容两端需空格，已确保即使在编码错误的情况下也可以正确解析样式。

在必要的情况下，可以使用块状注释，块状注释保持统一的缩进对齐。

原则上每个系列的样式都需要有一个注释，言简意赅的表明名称、用途、注意事项等。

通用

选择器

[强制] 如无必要，不得为 id、class 选择器添加类型选择器进行限定。

[建议] 选择器的嵌套层级应不大于 3 级，位置靠后的限定条件应尽可能精确。

属性缩写

[建议] 在可以使用缩写的情况下，尽量使用属性缩写。

[建议] 使用 border / margin / padding 等缩写时，应注意隐含值对实际数值的影响，确实需要设置多个方向的值时才使用缩写。

border / margin / padding 等缩写会同时设置多个属性的值，容易覆盖不需要覆盖的设定。如某些方向需要继承其他声明的值，则应该分开设置。

属性声明顺序

相关的属性声明应归为一组，建议按以下顺序进行声明：布局定位属性–>自身属性–>文本属性–>其他属性

布局定位属性：
包括：margin、padding、float（包括clear）、position（相应的top,right,bottom,left）、display、visibility、overflow

自身属性:
包括：width、height、background、border、border-radius;

文本属性：
包括：font、color、text-align、text-decoration、text-indent;

其他属性：
包括：list-style(列表样式)、vertical-vlign、cursor、z-index(层叠顺序) 、zoom、opacity、animation、transfrom、transition

选择器顺序

请综合考虑以下顺序依据：

从大到小（以选择器的范围为准）

从低到高（以等级上的高低为准）

从先到后（以结构上的先后为准）

从父到子（以结构上的嵌套为准）

!important

[建议] 尽量不使用 !important 声明。

[建议] 当需要强制指定样式且不允许任何场景覆盖时，通过标签内联和 !important 定义样式。

值与单位

文本

[强制] 文本内容必须用双引号包围。

数值

[强制] 当数值为 0 - 1 之间的小数时，省略整数部分的 0。

url()

[强制] url() 函数中的路径不加引号。
[建议] url() 函数中的绝对路径可省去协议名。

值为0

[强制] 值为 0 时须省略单位。

颜色

[强制] RGB颜色值必须使用十六进制记号形式 #rrggbb。不允许使用 rgb()。指定透明度时可以使用rgba，并且每个，后保留一个空格。

[强制] 颜色值可以缩写时，必须使用缩写形式；不允许使用命名色值；英文字符采用小写，如不用小写也需要保证同一项目内保持大小写一致。

[强制] 值为0-1之间的小数时，省略小数点前面的0

文字编排

字体族

[强制] font-family 属性中的字体族名称应使用字体的英文 Family Name，其中如有空格，须放置在引号中。

所谓英文 Family Name，为字体文件的一个元数据，常见名称如下：

字体名 操作系统 Family Name
宋体 (中易宋体) Windows SimSun
黑体 (中易黑体) Windows SimHei
微软雅黑 Windows Microsoft YaHei
微软正黑 Windows Microsoft JhengHei
华文黑体 Mac/iOS STHeiti
冬青黑体 Mac/iOS Hiragino Sans GB
文泉驿正黑 Linux WenQuanYi Zen Hei
文泉驿微米黑 Linux WenQuanYi Micro Hei

[强制] font-family 按「西文字体在前、中文字体在后」、「效果佳 (质量高/更能满足需求) 的字体在前、效果一般的字体在后」的顺序编写，最后必须指定一个通用字体族( serif / sans-serif )。

[强制] font-family 不区分大小写，但在同一个项目中，同样的 Family Name 大小写必须统一。

字号

[强制] 需要在 Windows 平台显示的中文内容，其字号应不小于 12px。

由于 Windows 的字体渲染机制，小于 12px 的文字显示效果极差、难以辨认。

字重

[强制] font-weight 属性必须使用数值方式描述。

解释：

CSS 的字重分 100 – 900 共九档，但目前受字体本身质量和浏览器的限制，实际上支持 400 和 700 两档，分别等价于关键词 normal 和 bold。

浏览器本身使用一系列启发式规则来进行匹配，在 =700 时匹配 Bold 字重。

但已有浏览器开始支持 =600 时匹配 Semibold 字重 (见此表)，故使用数值描述增加了灵活性，也更简短。

响应式

[强制]将媒体查询放在尽可能相关规则的附近。不要将他们打包放在一个单一样式文件中或者放在文档底部。

[强制] Media Query 如果有多个逗号分隔的条件时，应将每个条件放在单独一行中。

[建议] 尽可能给出在高分辨率设备 (Retina) 下效果更佳的样式。

兼容性

属性前缀

[强制] 带私有前缀的属性由长到短排列，按冒号位置对齐。

标准属性放在最后，按冒号对齐方便阅读，也便于在编辑器内进行多行编辑。

命名规则

选择器命名规则

1、由于ID选择器的唯一性导致不能重用、尽量选用Class选择器；

2、为了和js变量名区分，尽量采用’-‘作为分隔符，而不是用下划线 ‘—‘;

3、命名应该尽量简约而不失语义，不要使用没有语义的命名，不推荐使用拼音作为样式名

4、分类的命名方法：使用单个字母加 ‘-‘为前缀：
布局（grid）（.g-）；模块（module）（.m-）；元件（unit）（.ui-）；功能（function）（.f-）；状态（.z-）;js交互(.j-)

其中（.j-）前缀只用于js交互的选取元素，不用于样式命名；
所有的选择器都以以上的前缀开头，后面跟上后代选择器，这样能避免全局命名污染；
