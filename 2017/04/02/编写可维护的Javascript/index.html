编写可维护的Javascript之编程风格

发表于 2017-04-02 | 分类于 编写可维护的Javascript | 阅读次数
最近在看O'REILLY出版的Nicholas C.Zakas（《Javascript高级程序设计》、《Ajax高级程序设计》、《高性能Javascript》的作者）著的《编写可维护的Javascript》，里面提到的编码规范值得借鉴，故整理出一份自己的编码规范。遵循一份好的编程规范可以有效提升代码可读性，提高开发效率，便于代码维护。

基本格式化

缩进层级：

以四个空格字符为一个缩进层级；可以在各编辑器中配置敲入Tab键时插入4个space。

语句结尾：

尽管有赖于分析器的自动分号插入（ASI）机制，Javascript代码省略分号也是可以正常工作的，但是语句结尾不要省略分号。

行的长度：

将一行的长度限定在80个字符。

换行：

当一行长度达到单行最大字符数限制时，需要手动将一行拆成两行。通常是在运算符后(将运算符置于行尾，ASI就不会自作主张插入分号)换行，下一行增加两个层级（两个Tab）的缩进；当给变量赋值时，第二行位置应当和赋值运算符的位置保持对齐。

空行:

以下情景使用空行：

每个流程控制语句（if和for等）之前，添加空行
方法之间
方法中的局部变量和第一条语句之间
多行或单行注释之前
方法内的逻辑片段之间cherubic空行，提高可读性

命名：

计算机科学只存在两个难题：缓存失效和命名。————Phil Karlton

Javascript语言核心ECMAScript，遵照驼峰式大小写命名法，由小写字母开始，后续每个单词首字母都大写。
1、变量和函数：
以小驼峰命名法（Camel Case），变量命名以名词为前缀，函数命名以动词为前缀，避免无意义的命名
2、常量：
常量以大写字母和下划线命名，下划线用来分割单词
3、构造函数：
构造函数的命名遵照大驼峰命名法（Pascal Case）:由大写字母开始，后续每个单词首字母大写

直接量：

1、字符串
2、数字
3、null：　用来当作对象的占位符
以下场景使用null:

用来初始化一个变量，这个变量可能赋值为一个对象
用来和一个已经初始化的变量比较，这个变量可以是也可以不是一个对象
当函数的参数期望是对象时，用作参数传入
当函数的返回值期望是对象时，用作返回值输出

以下场景不应当使用null：

不要使用null来检测是否传入了某个参数
不要用null来检测一个未初始化的变量

4、undefined: 表示一个等待被赋值的变量
未声明的变量和已声明但未赋值的变量（值为undefined），typeof运算符的返回结果都是”undefined”;

5、对象直接量
定义对象直接量时，常在第一行包含左花括号，每一个属性的名值对都独占一行，并保持一个缩进，最后右花括号也独占一行
